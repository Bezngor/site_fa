# Clarifications: hero-section

## Question 1
Какое изображение следует использовать в качестве фонового для Hero-секции?

*Context:* В спецификации указано использование next/image, но путь к файлу не определен, что критично для сборки.

**Options:**
- A) Использовать заглушку (placeholder) из внешнего сервиса (например, /images/hero-bg.webp)
- B) Использовать абстрактный градиент в цветах бренда (#1B2A4A) вместо реального фото
- C) Оставить пустой блок с комментарием TODO для добавления ассета позже

**Answer:**
Использовать заглушку (placeholder) из внешнего сервиса (например, /images/hero-bg.webp)

## Question 2
Как должна вести себя кнопка CTA при клике?

*Context:* Требуется реализация плавного скролла (smooth scroll) к секции формы.

**Options:**
- A) Использовать стандартный anchor link с CSS scroll-behavior: smooth
- B) Реализовать программный скролл через window.scrollTo в 'use client' компоненте
- C) Простой переход по якорю без анимации

**Answer:**
Использовать стандартный anchor link с CSS scroll-behavior: smooth

## Question 3
Нужно ли накладывать затемняющий слой (overlay) на фоновое изображение?

*Context:* Для обеспечения читаемости белого текста на фоне изображения часто требуется полупрозрачный слой.

**Options:**
- A) Да, добавить полупрозрачный слой цвета #1B2A4A (opacity 60%)
- B) Нет, использовать изображение как есть, полагаясь на его контрастность
- C) Использовать градиент от темного к прозрачному

**Answer:**
Использовать градиент от темного к прозрачному

## Question 4
Какую структуру заголовка H1 использовать для SEO?

*Context:* Hero-секция содержит главный заголовок страницы, важно определить его наполнение из texts.ts.

**Options:**
- A) Единый заголовок из одной строки
- B) Разделение на основной заголовок и подзаголовок (H1 + Span/Subtext)
- C) Заголовок с выделением ключевого слова акцентным цветом #F59E0B

**Answer:**
Разделение на основной заголовок и подзаголовок (H1 + Span/Subtext)

## Question 5
Должна ли Hero-секция занимать всю высоту экрана (viewport height)?

*Context:* Это влияет на верстку и восприятие первого экрана пользователем.

**Options:**
- A) Да, использовать min-h-screen для полного заполнения экрана
- B) Нет, использовать фиксированные отступы (например, py-20 или py-32)
- C) Автоматическая высота в зависимости от контента

**Answer:**
Да, использовать min-h-screen для полного заполнения экрана
