# Спецификация: main-page-skeleton

## Контекст
Главная страница собирает все секции в единую структуру. На данном этапе создается каркас (скелет) приложения с навигацией и заглушками для контента.

## User Scenarios
1. **Посетитель:** Заходит на сайт, видит шапку профиля и может быстро перемещаться между разделами страницы с помощью якорных ссылок.
2. **Посетитель:** Скроллит страницу и видит логическую последовательность блоков (даже если они еще не заполнены финальным контентом).

## Functional Requirements
- **FR-01:** Импорт и рендер реализованных секций: Hero, Problems, HowWeWork.
- **FR-02:** Реализация заглушек (stubs) для остальных секций: Cases, Products, Results, About, Contact. Заглушки должны представлять собой пустые контейнеры с текстовой пометкой `TODO: [Название секции]`.
- **FR-03:** Присвоение уникальных ID каждой секции для работы якорных ссылок: `hero`, `problems`, `workflow`, `cases`, `products`, `results`, `about`, `contact`.
- **FR-04:** Реализация компонента `Header` с навигационным меню, содержащим ссылки на все 8 ID секций.
- **FR-05:** Навигация должна быть фиксированной (sticky) или находиться в верхней части страницы для обеспечения постоянного доступа к разделам.

## Non-Functional Requirements
- **NFR-01:** Плавная прокрутка: использование CSS свойства `scroll-behavior: smooth` в `globals.css` для всех современных браузеров.
- **NFR-02:** Семантическая верстка: использование тегов `<header>`, `<main>`, `<section>`.

## Boundaries (what is NOT included)
- Финальный контент и стилизация для секций-заглушек.
- Компонент `Footer` (будет добавлен на этапе финализации проекта).
- Сложные JS-анимации появления блоков.

## Acceptance Criteria
- [ ] Все 8 секций (включая заглушки) присутствуют в DOM и имеют корректные ID.
- [ ] Компонент `Header` отображается и содержит рабочие ссылки на все разделы.
- [ ] При клике на ссылку в меню происходит плавный переход к соответствующему блоку.
- [ ] Секции Hero, Problems и HowWeWork отображают свой начальный контент.
- [ ] Заглушки четко помечены текстом "TODO: [Название]".

## Open Questions
- *Вопросов нет. Все технические детали реализации скелета и навигации согласованы.*

### AI_NOTES
- Использовать `sticky top-0` для Header, чтобы навигация всегда была под рукой.
- Для заглушек использовать минимальную высоту (например, `min-h-[200px]`), чтобы визуально протестировать скролл.
- Убедиться, что ID в Header полностью совпадают с ID в компонентах секций.
