# Спецификация: problems-section

## Контекст
Секция подчеркивает боли клиентов для плавного перехода к предлагаемому решению. Она визуализирует основные трудности, с которыми сталкивается целевая аудитория, создавая контекст необходимости продукта.

## User Scenarios
1. **Посетитель:** Просматривает список проблем, узнает в них свои текущие трудности и продолжает скроллить к разделу с решением.

## Functional Requirements
- **FR-01:** Данные загружаются из массива строк в `lib/data/texts.ts` (от 4 до 6 пунктов).
- **FR-02:** Отображение каждого пункта в виде карточки с крупным порядковым номером.
- **FR-03:** Наличие общего заголовка секции (например, "Проблемы, которые мы решаем").
- **FR-04:** Наличие краткого текстового вступления (подзаголовка) перед списком проблем.

## Non-Functional Requirements
- **NFR-01:** Анимация появления: использование стандартных CSS-классов Tailwind (например, `animate-fade-in`) без использования тяжелых JS-библиотек для scroll-tracking.
- **NFR-02:** Цветовая схема: порядковые номера должны быть выполнены в акцентном цвете `#F59E0B` (amber).
- **NFR-03:** Адаптивность сетки (Grid):
    - Mobile (до 767px): 1 колонка.
    - Tablet/Desktop (от 768px и выше): 2 колонки.

## Boundaries (что НЕ включено)
- Сложные анимации на базе Framer Motion или GSAP.
- Интерактивные элементы (кнопки, ссылки внутри карточек).
- Раскрывающиеся списки (аккордеоны).

## Acceptance Criteria
- [x] Сетка адаптивна: 1 колонка на мобильных, 2 колонки на десктопе.
- [x] Текст полностью на русском языке.
- [x] Вместо иконок используются крупные порядковые номера в цвете `#F59E0B`.
- [x] Данные берутся из типизированного массива строк в `texts.ts`.
- [x] Секция содержит заголовок и вводное описание.
- [x] Применены базовые классы анимации Tailwind для появления контента.
