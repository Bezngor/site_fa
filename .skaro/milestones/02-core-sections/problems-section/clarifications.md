# Clarifications: problems-section

## Question 1
Какую иконографику следует использовать для отображения проблем в карточках?

*Context:* Спецификация упоминает 'Icons или номера', но не определяет библиотеку или стиль, что влияет на размер бандла и визуальную консистентность.

**Options:**
- A) Использовать стандартные SVG-иконки (например, Lucide React) для визуализации каждой боли
- B) Использовать только крупные порядковые номера в акцентном цвете (#F59E0B)
- C) Использовать простые кастомные SVG-маркеры (например, крестики или предупреждающие знаки)

**Answer:**
Использовать только крупные порядковые номера в акцентном цвете (#F59E0B)

## Question 2
Какова структура данных для проблем в файле texts.ts?

*Context:* Для реализации типизированного компонента нужно понимать, содержит ли каждая проблема только текст или также заголовок и описание.

**Options:**
- A) Массив строк (только краткое описание проблемы)
- B) Массив объектов с полями 'title' (заголовок) и 'description' (подробное описание)
- C) Массив объектов с полями 'id', 'text' и 'iconName'

**Answer:**
Массив строк (только краткое описание проблемы)

## Question 3
Как должна вести себя сетка (Grid) на разных разрешениях экрана?

*Context:* Количество колонок определяет визуальный баланс секции при 4-6 пунктах.

**Options:**
- A) 1 колонка на мобильных, 2 колонки на планшетах, 3 колонки на десктопе
- B) 1 колонка на мобильных, 2 колонки на всех экранах выше 768px
- C) Список в одну колонку с широкими карточками для всех разрешений

**Answer:**
1 колонка на мобильных, 2 колонки на всех экранах выше 768px

## Question 4
Какой тип анимации появления при скролле (NFR-01) предпочтителен?

*Context:* Конституция ограничивает использование тяжелых библиотек, поэтому нужно выбрать метод реализации.

**Options:**
- A) Простой CSS transition с использованием Intersection Observer (opacity + transform)
- B) Стандартные классы Tailwind 'animate-fade-in' (без JS-логики появления)
- C) Не использовать анимацию для сохранения минимального размера JS-бандла

**Answer:**
Стандартные классы Tailwind 'animate-fade-in' (без JS-логики появления)
