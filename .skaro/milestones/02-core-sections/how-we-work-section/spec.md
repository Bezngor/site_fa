# Спецификация: how-we-work-section

## Контекст
Описание процесса взаимодействия с клиентом для выстраивания доверия и прозрачности. Согласно ADR-006, компонент реализуется как Server Component.

## User Scenarios
1. **Посетитель:** Просматривает секцию и видит четкую последовательность действий (timeline), понимая, что произойдет после обращения в компанию.

## Functional Requirements
- FR-01: Данные для шагов (заголовки и описания) должны импортироваться из массива в `lib/data/texts.ts`.
- FR-02: Визуализация в виде вертикального таймлайна с соединительными линиями между этапами.
- FR-03: Каждый шаг должен содержать порядковый номер внутри круга (стиль минимализм).
- FR-04: Обязательные этапы:
    1. Аудит
    2. Проектирование
    3. Разработка
    4. Внедрение
    5. Поддержка

## Non-Functional Requirements
- NFR-01: Максимальная вложенность DOM-элементов — 3 уровня (согласно Constitution).
- NFR-02: Цветовая палитра: основной темно-синий (#1B2A4A) для текста/линий и акцентный янтарный (#F59E0B) для номеров шагов.
- NFR-03: Адаптивность: на мобильных устройствах отображается полный текст описаний (стандартный скролл страницы).

## Boundaries (what is NOT included)
- Сложные JS-анимации появления элементов.
- Интерактивное переключение вкладок или аккордеоны.
- Использование сторонних библиотек иконок (используются только CSS/номера).

## Acceptance Criteria
- [ ] Секция корректно отображает все 5 этапов из Q&A.
- [ ] Визуально реализована вертикальная линия, соединяющая круги с номерами.
- [ ] Текст описаний читаем на мобильных устройствах без обрезки.
- [ ] Все шаги визуально равнозначны (нет выделения "текущего" этапа).
- [ ] Компонент является Server Component и не использует 'use client'.

## AI_NOTES
- Для реализации вертикальной линии таймлайна использовать псевдоэлементы `:before` или `:after` у контейнера списка или Border-left с отступом.
- Номера шагов стилизовать через Tailwind: `flex items-center justify-center rounded-full border-2`.
- Убедиться, что в `texts.ts` структура объекта соответствует: `{ steps: [{ title: string, desc: string }] }`.
