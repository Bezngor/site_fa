# Clarifications: how-we-work-section

## Question 1
Какую структуру визуализации процесса (timeline) следует использовать?

*Context:* От этого зависит выбор CSS-сетки и адаптивность секции на мобильных устройствах.

**Options:**
- A) Вертикальный список с иконками и соединительными линиями (классический таймлайн)
- B) Горизонтальная сетка (карточки) с нумерацией (1, 2, 3...)
- C) Зигзагообразный макет (текст слева/справа) для десктопа, переходящий в стек на мобильных

**Answer:**
Вертикальный список с иконками и соединительными линиями (классический таймлайн)

## Question 2
Какие именно этапы работы должны быть отображены в секции?

*Context:* Данные должны быть определены в lib/data/texts.ts согласно архитектуре проекта.

**Options:**
- A) Стандартный цикл: Аудит -> Проектирование -> Разработка -> Внедрение -> Поддержка
- B) Упрощенный цикл: Заявка -> Обсуждение -> Реализация -> Результат
- C) Индивидуальный набор шагов (нужно предоставить тексты)

**Answer:**
Стандартный цикл: Аудит -> Проектирование -> Разработка -> Внедрение -> Поддержка

## Question 3
Требуется ли визуальное выделение текущего или ключевого этапа?

*Context:* Это влияет на сложность типизации данных в texts.ts и стилизацию компонентов.

**Options:**
- A) Нет, все шаги визуально равнозначны
- B) Да, выделить первый и последний шаги акцентным цветом (#F59E0B)
- C) Да, добавить специальный маркер 'Результат' для финального этапа

**Answer:**
Нет, все шаги визуально равнозначны

## Question 4
Должны ли шаги содержать иконки или изображения?

*Context:* Использование иконок потребует подключения библиотеки (например, Lucide) или подготовки SVG.

**Options:**
- A) Только порядковые номера в кругах (минимализм)
- B) Стандартные SVG иконки для каждого этапа
- C) Не использовать графику, только заголовки и краткое описание

**Answer:**
Только порядковые номера в кругах (минимализм)

## Question 5
Как обрабатывать длинные описания этапов в мобильной версии?

*Context:* Ограничение по вложенности (max 3 уровня) и бюджет бандла требуют простых решений.

**Options:**
- A) Отображать полный текст (скролл страницы)
- B) Использовать фиксированную высоту карточек с обрезкой текста
- C) Скрывать подробное описание, оставляя только заголовки на мобильных

**Answer:**
Отображать полный текст (скролл страницы)
