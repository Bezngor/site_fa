# AI_NOTES — Stage 1: Создание библиотеки UI-компонентов

## Что было сделано
- Реализован компонент `Button` с поддержкой вариантов `primary`/`secondary`, состояния загрузки (`isLoading`) и стандартных HTML-атрибутов.
- Реализован компонент `FormInput` с поддержкой `input`/`textarea`, валидацией (пропс `error`) и `forwardRef` для интеграции.
- Создан `index.ts` для экспорта компонентов.

## Почему выбран этот подход
- Использованы стандартные Tailwind-классы для обеспечения консистентности с палитрой проекта (#1B2A4A, #F59E0B).
- `forwardRef` в `FormInput` необходим для будущей интеграции с библиотеками управления формами.
- Компоненты не имеют внешних отступов (margin), что соответствует требованию NFR-02.

| Файл | Действие | Описание |
|---|---|---|
| `components/ui/Button.tsx` | создано | Кнопка с вариантами и состоянием загрузки |

## Риски и ограничения
- `textarea` ограничена `resize-y` для сохранения целостности дизайна.
- Использование `'use client'` оправдано интерактивностью компонентов (состояния hover, focus, loading).
