# Clarifications: ui-components

## Question 1
Какие именно визуальные варианты (variants) должны быть у компонента Button?

*Context:* Это определяет структуру пропсов и набор Tailwind-классов для обеспечения консистентности дизайна во всех секциях.

**Options:**
- A) Primary (amber background, dark text) и Secondary (transparent, border amber)
- B) Primary (dark navy background, white text) и Ghost (no background, dark text)
- C) Только один вариант (Primary) согласно цветовой палитре #F59E0B

**Answer:**
Primary (amber background, dark text) и Secondary (transparent, border amber)

## Question 2
Как должна обрабатываться валидация в компоненте FormInput?

*Context:* Необходимо решить, будет ли компонент сам отображать ошибки Zod или это ответственность родительской формы.

**Options:**
- A) FormInput принимает пропс error (string) и отображает красный текст под полем
- B) FormInput только стилизует границу красным при наличии ошибки, текст ошибки выводит форма
- C) Валидация не отражается на стиле UI-компонента (упрощенный вариант)

**Answer:**
FormInput принимает пропс error (string) и отображает красный текст под полем

## Question 3
Нужна ли поддержка различных типов ввода для FormInput (textarea, tel, email)?

*Context:* Контактная форма может содержать многострочные сообщения или специфические маски ввода.

**Options:**
- A) Единый компонент с пропсом type (input или textarea)
- B) Раздельные компоненты: FormInput и FormTextarea
- C) Только стандартный input type='text', остальные поля делать нативными

**Answer:**
Единый компонент с пропсом type (input или textarea)

## Question 4
Должен ли компонент Button поддерживать состояние загрузки (loading state)?

*Context:* При отправке формы через Formspree пользователю важно видеть фидбек о процессе.

**Options:**
- A) Да, добавить пропс isLoading, который отключает кнопку и показывает спиннер
- B) Нет, достаточно стандартного атрибута disabled во время отправки
- C) Не нужно, обработка состояния будет только на уровне текста кнопки

**Answer:**
Да, добавить пропс isLoading, который отключает кнопку и показывает спиннер

## Question 5
Какую стратегию внешних отступов (margins) использовать для UI-компонентов?

*Context:* Для обеспечения переиспользуемости важно решить, как компоненты позиционируются в макете.

**Options:**
- A) Компоненты не имеют внешних отступов, отступы задаются родителем
- B) Компоненты принимают опциональный пропс className для передачи margin
- C) Фиксированные отступы внутри компонентов по сетке Tailwind

**Answer:**
Компоненты не имеют внешних отступов, отступы задаются родителем
