# Спецификация: ui-components

## Контекст
Создание библиотеки базовых UI-компонентов для обеспечения единообразия интерфейса и исключения дублирования кода (DRY). Компоненты используются во всех секциях лендинга.

## User Scenarios
1. **Разработчик:** Импортирует `Button` или `FormInput` в компоненты секций для создания интерактивных элементов и форм обратной связи.

## Functional Requirements
- **FR-01: Button (Кнопка)**
    - Поддержка двух визуальных вариантов:
        - `Primary`: фон amber (#F59E0B), темный текст.
        - `Secondary`: прозрачный фон, граница (border) цвета amber.
    - Пропс `isLoading`: при значении `true` кнопка становится неактивной (`disabled`) и отображает спиннер.
    - Пропс `type`: поддержка типов `button`, `submit`, `reset`.
- **FR-02: FormInput (Поле ввода)**
    - Поддержка различных типов ввода через пропс `as`: `input` или `textarea`.
    - Поддержка стандартных атрибутов HTML (`type="tel"`, `type="email"`, `placeholder` и т.д.).
    - Отображение `label` над полем.
    - Валидация: прием пропса `error` (строка). Если пропс передан, отображается красный текст ошибки под полем, а граница поля подсвечивается красным.

## Non-Functional Requirements
- **NFR-01: Tailwind only:** Стилизация исключительно через классы Tailwind CSS 4.
- **NFR-02: No Margins:** Компоненты не должны иметь внешних отступов (`margins`). Позиционирование задается родительским контейнером.
- **NFR-03: Accessibility:** Использование семантических тегов и поддержка состояний `:focus-visible`.

## Boundaries (what is NOT included)
- Сложные компоненты вроде Modal (реализуются отдельно при необходимости).
- Библиотеки иконок (использовать SVG или простые символы, если не указано иное).

## Acceptance Criteria
- [x] Рефакторинг существующих секций с использованием новых UI-компонентов.
- [ ] Компонент `Button` корректно переключает состояния (default, loading, disabled).
- [ ] Компонент `FormInput` корректно отображает ошибки валидации и меняет тип элемента (input/textarea).
- [ ] Отсутствуют жестко закодированные внешние отступы внутри UI-компонентов.
- [ ] Код соответствует стандартам проекта (TypeScript, лимиты строк).

**Ключевые решения:**
1. **Button:** Реализовать через `cva` (class-variance-authority) или аналогичный подход для управления вариантами `primary`/`secondary`. Для спиннера использовать SVG с анимацией `animate-spin`.
2. **FormInput:** Использовать полиморфный подход или простое условие для рендеринга `input` vs `textarea`. Обязательно прокидывать `ref` через `forwardRef` для интеграции с библиотеками форм (например, React Hook Form, если потребуется в будущем).
3. **Стили:** Цвета брать из конфигурации Tailwind (amber, primary navy).

**Риски:**
- При использовании `textarea` нужно предусмотреть корректную обработку высоты (resize-none или auto-resize). По умолчанию ограничимся `resize-y`.
