# AI_NOTES — Stage 1: Конфигурация Next.js для статического экспорта и изображений

## What was done
- Проверена конфигурация Next.js в `next.config.js`. Убеждены, что установлены `output: 'export'` и `images.unoptimized: true`.
- Создан файл `AI_NOTES.md` с документацией по конфигурации и обоснованием использования `unoptimized`.

## Why this approach
- `output: 'export'` необходим для генерации статических файлов (HTML/CSS/JS) на этапе сборки, что позволяет деплоить сайт на любой статический хостинг (Vercel, Netlify, S3 и т.д.) без серверного рантайма Next.js.
- `images.unoptimized: true` отключает встроенную оптимизацию изображений on-demand, которая требует серверного API (Image Optimization API). При статическом экспорте эта функциональность недоступна, поэтому изображения должны быть предварительно оптимизированы и помещены в `public/`. Это также предотвращает попытки Next.js генерировать оптимизированные версии во время сборки, что может вызвать ошибки.

## Risks and limitations
- Без on-demand оптимизации ответственность за размер и формат изображений (WebP) лежит на разработчике. Необходимо вручную контролировать вес файлов в `public/images/` для соблюдения лимита bundle size и метрик LCP/CLS.
- При использовании `next/image` с `unoptimized: true` атрибуты `width`/`height` обязательны для предотвращения CLS, если не используется `fill`. В данном проекте для Hero используется `fill`, а для кейсов — фиксированные пропорции контейнеров.

## Invariant compliance
- [ ] Статическая генерация (SSG) — соблюдено: `output: 'export'` обеспечивает полный статический экспорт.
- [ ] Изображения через `next/image` — соблюдено: компоненты используют `next/image`, а `unoptimized: true` совместим с экспортом.
- [ ] Bundle size <150 KB — не проверяется на этом этапе, но отключение on-demand оптимизации не влияет на JS-бандл, только на вес статических ассетов.

## How to verify
1. Проверить содержимое `next.config.js`: убедиться, что есть `output: 'export'` и `images.unoptimized: true`.
2. Выполнить `npm run build` (или `next build`). Сборка должна завершиться без ошибок и сгенерировать папку `out/` со статическими файлами.
3. Убедиться, что в `out/` присутствуют изображения из `public/` (например, `out/images/hero.webp`).
4. Проверить, что в сгенерированном HTML нет ссылок на Image Optimization API (например, `/_next/image`).
