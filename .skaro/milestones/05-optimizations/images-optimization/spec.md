# Спецификация: images-optimization

## Контекст

Оптимизация графики для LCP/CLS при статическом экспорте Next.js (`output: 'export'`). Растровые ассеты заранее кладутся в `public/images/` в формате WebP; встроенный on-demand Image Optimization в рантайме для экспорта не используется — включён `images.unoptimized` (см. `AI_NOTES.md`).

## User Scenarios

1. **Посетитель:** Hero загружается с приоритетом; изображения кейсов подгружаются при раскрытии блока; нет скачков вёрстки из‑за неизвестных размеров растра там, где заданы контейнеры и данные.

## Functional Requirements

- **FR-01: `next/image` для растра:** Hero и изображения в кейсах выводятся через `next/image`.
- **FR-02: Hero (LCP):** Для фона Hero — `priority`, `fetchPriority="high"`, `fill` и осмысленный `alt` из данных.
- **FR-03: Адаптивность (`sizes`):** Для кейсов — `sizes="(max-width: 768px) 100vw, (max-width: 1200px) 50vw, 33vw"`; для Hero — `100vw`.
- **FR-04: Формат растра:** Файлы в `public/` — WebP (ADR-001).
- **FR-05: Источники ассетов:** Пути к растру задаются в данных (`texts.hero.backgroundSrc`, `cases[].image` и т.д.); обязательного использования picsum и внешних URL нет.
- **FR-06: Кейсы (inline expand):** Изображение кейса рендерится только в раскрытом состоянии; у `Image` — `loading="lazy"`.

## Исключения и границы (явно)

- **ProductLine / карточки продуктов:** Визуал — иконки **Lucide** (`ProductCard`), растровый `next/image` не требуется и не является частью DoD этого milestone.
- **About:** Секция без фотографий (только текст и иконки Lucide). Не добавлять растровые изображения из устаревших сценариев Stage 4 без отдельного ТЗ.
- **Централизованный `lib/data/images.ts`:** Не обязателен, если пути и метаданные уже заданы в `texts` / `cases` и типах.

## Non-Functional Requirements (ориентиры)

- **NFR-01:** LCP — ориентир \< 2.5 s (проверка: Lighthouse/Field Data на целевом хостинге).
- **NFR-02:** CLS — ориентир \< 0.1; для `fill` — фиксированные пропорции контейнера; в данных кейсов хранятся `imageWidth` / `imageHeight` для согласованности и возможных расширений.
- **NFR-03:** Лимит JS-бандла 150 KB gzipped — ориентир из ADR/плана; фактическую величину проверять после `npm run build` (не смешивать с весом статики/WebP).

## Что не входит

- Генерация уникальных изображений в рамках задачи.
- AVIF (по соглашению — WebP).
- Обязательный README в корне репозитория; при договорённости команды — краткий подпункт «Изображения» или ссылка на `AI_NOTES.md`.

## Acceptance Criteria (проверка по факту)

- [ ] Растр Hero и кейсов использует `next/image`; Hero с `priority` и `fetchPriority="high"`.
- [ ] Файлы `public/images/hero.webp`, `public/images/cases/case*.webp` присутствуют в репозитории (или явно описана замена путей).
- [ ] `npm run build` завершается без ошибок (статический экспорт через `next build`).
- [ ] Кейсы: картинка не запрашивается до раскрытия аккордеона (условный рендер + lazy).
- [ ] ProductLine без растровых карточек; About без фото — согласно исключениям выше.
- [ ] NFR/Lighthouse/CWV — по чеклисту в `AI_NOTES.md` перед релизом, без автоматического «[x]» без замера.
