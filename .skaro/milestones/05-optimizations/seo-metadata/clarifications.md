# Clarifications: seo-metadata

**Примечание:** ответы фиксируют архитектурные решения (layout vs page, статические robots/sitemap и т.д.). Примеры URL в вариантах (в т.ч. Vercel) — иллюстративные; **канонический prod-origin и копирайт meta** — как в `app/layout.tsx`, `.env.example` и документе позиционирования, а не обязательный текст из старого шаблона Skaro.

## Question 1
Где именно должны быть определены метаданные: в layout.tsx или page.tsx?

*Context:* Next.js позволяет определять метаданные на обоих уровнях, но для одностраничного лендинга важно избежать дублирования или конфликтов.

**Options:**
- A) Определить общие метаданные в layout.tsx, а специфичные для страницы (title, description) в page.tsx через generateMetadata
- B) Разместить все метаданные (включая OG и иконки) только в layout.tsx, так как страница всего одна
- C) Использовать статическую переменную metadata в page.tsx без использования generateMetadata

**Answer:**
Разместить все метаданные (включая OG и иконки) только в layout.tsx, так как страница всего одна

## Question 2
Какой базовый URL использовать для формирования абсолютных путей к OG-изображениям?

*Context:* Для корректной работы Open Graph тегов (og:image) требуются абсолютные URL, которые зависят от финального домена деплоя.

**Options:**
- A) Использовать переменную окружения NEXT_PUBLIC_SITE_URL (например, https://site-factoryall.vercel.app)
- B) Захардкодить основной домен в конфигурации метаданных
- C) Использовать относительные пути (может привести к ошибкам отображения превью в соцсетях)

**Answer:**
Использовать переменную окружения `NEXT_PUBLIC_SITE_URL` (в репозитории для prod в `.env.example` задано `https://factoryall.ru`; на превью — свой origin).

## Question 3
Нужно ли добавлять поддержку альтернативных языков или локалей в метаданные?

*Context:* Конституция проекта указывает на использование только русского языка, но SEO-стандарты часто требуют явного указания локали.

**Options:**
- A) Явно указать locale: 'ru_RU' в объекте openGraph
- B) Не указывать локаль, полагаясь на атрибут lang='ru' в теге html
- C) Добавить массив alternate languages (не рекомендуется для текущего объема)

**Answer:**
Явно указать locale: 'ru_RU' в объекте openGraph

## Question 4
Какую стратегию выбрать для генерации robots.txt и sitemap.xml?

*Context:* В дереве файлов уже присутствуют статические robots.txt и sitemap.xml, но Next.js 15 предлагает программные способы их генерации.

**Options:**
- A) Оставить текущие статические файлы в папке public как есть
- B) Перейти на программную генерацию через robots.ts и sitemap.ts в папке app для автоматизации путей
- C) Удалить эти файлы и управлять индексацией только через мета-теги robots в коде

**Answer:**
Оставить текущие статические файлы в папке public как есть

## Question 5
Какие дополнительные поля Open Graph необходимо включить помимо базовых?

*Context:* Для качественного отображения ссылки в мессенджерах (Telegram, WhatsApp) и соцсетях могут потребоваться уточняющие теги.

**Options:**
- A) Только базовые: title, description, image
- B) Расширенный набор: og:type ('website'), og:site_name, и twitter:card ('summary_large_image')
- C) Минимальный набор без Twitter-карточек

**Answer:**
Расширенный набор: og:type ('website'), og:site_name, и twitter:card ('summary_large_image')
