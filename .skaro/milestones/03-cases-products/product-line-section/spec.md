# Спецификация: product-line-section

## Контекст
Представление основной продуктовой линейки компании в виде сетки карточек для быстрого ознакомления с ключевыми предложениями.

## User Scenarios
1. **Посетитель:** Просматривает доступные продукты в сетке (grid), сравнивает их характеристики по списку фич и переходит к оформлению заявки через кнопку действия.

## Functional Requirements
- **FR-01:** Данные загружаются из массива в `lib/data/products.ts`.
- **FR-02:** Отображение ровно 3 продуктов в один ряд на десктопных устройствах (адаптивность: 1 колонка на мобильных).
- **FR-03:** Каждая карточка должна содержать:
    - Изображение/Иконку (абстрактная иконка в цвете `#1B2A4A` при отсутствии фото).
    - Название продукта.
    - Список ключевых особенностей (features) в виде маркированного списка (bullet points).
    - Индивидуальную кнопку действия («Выбрать» или «Заказать»).
- **FR-04:** Кнопка в карточке должна прокручивать страницу к секции формы обратной связи (CTA).

## Non-Functional Requirements
- **NFR-01:** Максимальная длина файла компонента — 40 строк (использовать подкомпоненты при необходимости).
- **NFR-02:** Использование цветовой палитры: основной `#1B2A4A`, акцентный `#F59E0B`.
- **NFR-03:** Изображения оптимизированы через `next/image`.

## Boundaries (what is NOT included)
- Страницы с подробным описанием отдельных продуктов (детальные страницы).
- Фильтрация или сортировка продуктов.
- Корзина или функционал электронной коммерции.

## Acceptance Criteria
- [ ] Секция успешно интегрирована в `app/page.tsx`.
- [ ] На десктопе отображается ровно 3 карточки в ряд.
- [ ] Фичи продукта отображаются корректным маркированным списком.
- [ ] Каждая карточка имеет кнопку, ведущую к форме заказа.
- [ ] При отсутствии реальных фото используются брендированные иконки (цвет `#1B2A4A`).
- [ ] Верстка полностью адаптивна (мобильная версия проверена).
