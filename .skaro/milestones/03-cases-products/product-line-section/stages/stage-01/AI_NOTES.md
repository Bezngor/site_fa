# AI_NOTES — Этап 1: Подготовка данных продуктов

## Что было сделано
- **`types/index.ts`**: Добавлен новый интерфейс `Product` для типизации карточек продуктов. Удален устаревший интерфейс `ProductCategory`.
- **`lib/data/products.ts`**: Файл полностью переписан. Вместо старого массива `productCategories` создан и экспортирован новый массив `products`, содержащий 3 объекта в соответствии с новым интерфейсом `Product`. Данные заполнены на русском языке, включая названия, списки характеристик и тексты кнопок.

## Почему такой подход
- **Типобезопасность**: Создание интерфейса `Product` и его использование в файле с данными гарантирует консистентность структуры данных и помогает избежать ошибок на этапе разработки и сборки.
- **Централизация данных**: Хранение данных о продуктах в отдельном файле (`lib/data/products.ts`) соответствует архитектурному решению ADR-003 и упрощает дальнейшее управление контентом.
- **Гибкость иконок**: Использование строковых имен (`'Cpu'`, `'CalendarClock'`) в качестве заполнителей для иконок позволяет на следующем этапе легко интегрировать любую библиотеку иконок (например, `lucide-react`), динамически отображая их по имени. Это более гибко, чем жестко прописывать пути к файлам.

## Файлы созданы / изменены
| Файл | Действие | Описание |
|---|---|---|
| `lib/data/products.ts` | изменен | Заменён старый массив `productCategories` на новый, типизированный массив `products` с данными для трех продуктовых карточек. |
| `types/index.ts` | изменен | Добавлен интерфейс `Product` для описания структуры продукта и удален неиспользуемый `ProductCategory`. |

## Риски и ограничения
- **Зависимость от иконок**: Реализация отображения иконок на следующем этапе будет зависеть от наличия библиотеки, способной рендерить иконки по строковому имени. Это было учтено в плане (использование `lucide-react`).

## Соответствие инвариантам
- **ADR-003: Хранение данных в статических TS/JSON файлах**: Соблюдено. Данные для продуктов хранятся в статическом, типизированном TS-файле.
- **Именование**: Соблюдено. Имена переменных (`products`) и интерфейсов (`Product`) соответствуют стандартам.
- **Русский язык**: Соблюдено. Все данные, предназначенные для отображения, написаны на русском языке.
