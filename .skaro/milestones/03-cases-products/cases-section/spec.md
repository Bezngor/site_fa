# Specification: cases-section (Секция «Наши проекты»)

## Context
Секция предназначена для демонстрации экспертности компании через реальные примеры выполненных работ. Она должна убедить потенциального клиента в надежности и качестве услуг, предоставляя конкретные цифры и результаты.

## User Scenarios
1. **Просмотр портфолио:** Пользователь прокручивает страницу до секции кейсов, чтобы увидеть масштаб и сложность решаемых задач.
2. **Изучение деталей:** Пользователь кликает на интересующий проект, чтобы узнать подробности реализации (проблему, стек, результат) без перехода на другую страницу.
3. **Оценка результатов:** Пользователь ищет конкретные метрики (цифры) в кейсе, чтобы сопоставить их со своими бизнес-задачами.

## Functional Requirements
- **FR-01: Отображение списка:** Все кейсы отображаются в виде вертикального списка (один под другим).
- **FR-02: Механизм развертывания (Аккордеон):** При клике на карточку кейса контент плавно разворачивается вниз, сдвигая нижестоящие элементы.
- **FR-03: Состав данных (краткий вид):** Название проекта, краткое описание, основное изображение.
- **FR-04: Состав данных (развернутый вид):** 
    - Проблема (описание болей клиента).
    - Решение (что было сделано).
    - Технологический стек (список технологий).
    - Результат в цифрах (конкретные KPI).
    - Цитата клиента (отзыв).
- **FR-05: Статичность изображений:** Изображения в кейсах являются иллюстративными, клик по ним не открывает модальных окон или лайтбоксов.

## Non-Functional Requirements
- **NFR-01: Анимация:** Плавный переход (transition) при раскрытии аккордеона (не более 300мс).
- **NFR-02: Адаптивность:** Корректное отображение на мобильных устройствах (текст не должен выходить за границы экрана).
- **NFR-03: Производительность:** Использование `next/image` для оптимизации веса изображений кейсов.

## Boundaries (what is NOT included)
- Фильтрация кейсов по категориям или тегам.
- Поиск по кейсам.
- Отдельные страницы для каждого кейса (все данные подгружаются статически в рамках одной страницы).
- Слайдер или карусель (используется только вертикальный список).

## Acceptance Criteria
- [ ] Секция отображает все кейсы из статического файла данных.
- [ ] При клике на заголовок или карточку кейса открывается блок с подробной информацией (Аккордеон).
- [ ] Одновременно может быть открыт только один кейс (или несколько, если это не мешает UX — на усмотрение реализации, но аккордеон должен работать стабильно).
- [ ] В развернутом состоянии присутствуют все 5 обязательных полей: Проблема, Решение, Стек, Результат, Цитата.
- [ ] Изображения оптимизированы и не вызывают сдвигов верстки (Layout Shift).
- [ ] Верстка соответствует цветовой палитре: основной #1B2A4A, акцент #F59E0B.
