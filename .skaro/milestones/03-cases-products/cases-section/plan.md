# План реализации: cases-section (Секция «Наши проекты»)

## Обзор
Задача: Реализовать секцию кейсов с механизмом аккордеона для отображения детальной информации о проектах. Секция должна демонстрировать экспертность компании через реальные примеры с конкретными метриками и результатами.

Архитектурный подход: Server Component для статического контента + 'use client' для интерактивного аккордеона. Данные хранятся в статическом TypeScript файле, изображения оптимизированы через next/image.

---

## Stage 1: Структура данных и типы для кейсов

**Цель:** Создать типизированную структуру данных для кейсов с полным набором полей (Проблема, Решение, Стек, Результат, Цитата) и наполнить реальными данными.

**Зависит от:** нет

**Входные данные:**
- Спецификация (FR-04: состав данных)
- Существующий `lib/data/cases.ts` (для рефакторинга)
- `types/index.ts` (для добавления типов)

**Выходные данные:**
- `types/index.ts` — добавлены типы `Case`, `CaseDetails`, `CaseResult`, `CaseTestimonial`
- `lib/data/cases.ts` — обновлен с полной структурой данных (минимум 3 кейса с реальными данными)

**Definition of Done:**
- [ ] Типы `Case` включают все обязательные поля: id, title, shortDescription, image, details (problem, solution, technologies, results, testimonial)
- [ ] Тип `CaseResult` содержит metric (название метрики) и value (значение с единицами измерения)
- [ ] Тип `CaseTestimonial` содержит text, author, position, company
- [ ] Массив `cases` экспортирован из `lib/data/cases.ts` и содержит минимум 3 полностью заполненных кейса
- [ ] Все строковые данные на русском языке
- [ ] Технологический стек представлен массивом строк (например: ['Next.js', 'TypeScript', 'PostgreSQL'])
- [ ] Результаты содержат конкретные цифры (например: '50% снижение времени загрузки', '3x рост конверсии')

**Риски:**
- Недостаточно реальных данных для наполнения — решение: использовать правдоподобные примеры из типичных проектов автоматизации производства

## Stage 2: UI-компоненты для аккордеона и карточки кейса

**Цель:** Создать переиспользуемые UI-компоненты для отображения кейса в свернутом и развернутом виде с плавной анимацией раскрытия.

**Зависит от:** Stage 1

**Входные данные:**
- `types/index.ts` (типы Case)
- Спецификация (NFR-01: анимация, FR-02: механизм аккордеона)
- `tailwind.config.ts` (цветовая палитра)

**Выходные данные:**
- `components/sections/CaseCard.tsx` — клиентский компонент карточки кейса с состоянием открыт/закрыт
- `components/sections/CaseDetails.tsx` — серверный компонент для отображения детальной информации (Проблема, Решение, Стек, Результат, Цитата)
- `components/ui/AccordionContent.tsx` — переиспользуемый компонент для анимированного контента аккордеона

**Definition of Done:**
- [ ] `CaseCard` — клиентский компонент ('use client') с локальным состоянием `isOpen`
- [ ] При клике на карточку переключается состояние `isOpen` и плавно разворачивается контент
- [ ] Анимация раскрытия использует Tailwind transitions (duration-300, ease-in-out)
- [ ] В свернутом виде отображается: изображение (next/image), название, краткое описание, кнопка "Подробнее"
- [ ] В развернутом виде дополнительно отображается `CaseDetails` с 5 блоками: Проблема, Решение, Стек, Результат, Цитата
- [ ] `CaseDetails` — серверный компонент, принимает `details` из типа `Case`
- [ ] Блок "Результат" отображает метрики в виде сетки с выделением цифр акцентным цветом (#F59E0B)
- [ ] Блок "Цитата" оформлен с кавычками и информацией об авторе (имя, должность, компания)
- [ ] Технологический стек отображается в виде тегов/бейджей
- [ ] Изображения оптимизированы через next/image с sizes и priority для первого кейса
- [ ] Адаптивная верстка: на мобильных устройствах текст не выходит за границы, изображение масштабируется

**Риски:**
- Сложность анимации раскрытия с динамической высотой — решение: использовать `grid-rows-[0fr]` → `grid-rows-[1fr]` или `max-height` с transition
- Layout Shift при загрузке изображений — решение: указать `width` и `height` для next/image

## Stage 3: Интеграция секции CasesSection и финальная сборка

**Цель:** Обновить компонент `CasesSection` для использования новых данных и компонентов, обеспечить корректное отображение вертикального списка кейсов с аккордеоном.

**Зависит от:** Stage 1, Stage 2

**Входные данные:**
- `components/sections/CasesSection.tsx` (существующий файл для рефакторинга)
- `components/sections/CaseCard.tsx`
- `lib/data/cases.ts`
- `app/page.tsx` (для проверки интеграции)

**Выходные данные:**
- `components/sections/CasesSection.tsx` — обновленный серверный компонент секции
- `components/sections/index.ts` — обновлен экспорт (если требуется)

**Definition of Done:**
- [ ] `CasesSection` — серверный компонент, импортирует данные из `lib/data/cases.ts`
- [ ] Секция содержит заголовок "Наши проекты" и подзаголовок с описанием
- [ ] Кейсы отображаются в виде вертикального списка (один под другим) с использованием `CaseCard`
- [ ] Каждый кейс передается в `CaseCard` как пропс
- [ ] Секция имеет id="cases" для якорной навигации
- [ ] Применены стили секции: padding, background (если требуется), max-width контейнера
- [ ] Цветовая палитра соблюдена: основной #1B2A4A, акцент #F59E0B
- [ ] На странице `app/page.tsx` секция `CasesSection` корректно интегрирована между другими секциями
- [ ] Визуальная проверка: все 3 кейса отображаются, аккордеон работает, анимация плавная

**Риски:**
- Конфликт состояния при одновременном открытии нескольких кейсов — решение: каждый `CaseCard` управляет своим состоянием независимо (допустимо открытие нескольких кейсов согласно спецификации)

## Stage 4: E2E тесты для секции кейсов

**Цель:** Создать автоматизированные тесты для проверки функциональности секции кейсов (отображение, раскрытие аккордеона, наличие всех данных).

**Зависит от:** Stage 3

**Входные данные:**
- `e2e/sections.spec.ts` (существующий файл для расширения)
- `e2e/helpers/selectors.ts` (для добавления селекторов)
- `playwright.config.ts`

**Выходные данные:**
- `e2e/cases-section.spec.ts` — новый файл с тестами для секции кейсов
- `e2e/helpers/selectors.ts` — добавлены селекторы для элементов кейсов

**Definition of Done:**
- [ ] Тест "Секция кейсов отображается и содержит все кейсы" — проверяет наличие секции и количество карточек (минимум 3)
- [ ] Тест "Клик на карточку кейса раскрывает детали" — кликает на первый кейс, проверяет появление блоков Проблема, Решение, Стек, Результат, Цитата
- [ ] Тест "Детали кейса содержат все обязательные поля" — проверяет наличие текста в каждом из 5 блоков
- [ ] Тест "Изображения кейсов загружаются корректно" — проверяет атрибуты src и alt у изображений
- [ ] Тест "Аккордеон работает для всех кейсов" — последовательно открывает и закрывает каждый кейс
- [ ] Селекторы добавлены в `e2e/helpers/selectors.ts`: `CASES_SECTION`, `CASE_CARD`, `CASE_DETAILS`, `CASE_PROBLEM`, `CASE_SOLUTION`, `CASE_TECH_STACK`, `CASE_RESULTS`, `CASE_TESTIMONIAL`
- [ ] Тесты проходят успешно при запуске `npm run test:e2e`

**Риски:**
- Нестабильность тестов из-за анимации — решение: использовать `waitForSelector` с timeout и проверять visibility после завершения transition
