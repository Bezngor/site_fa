## Риски и ограничения
- **Жёсткая привязка к тексту ошибки:** Тест пустой формы проверяет только сообщение `'Имя должно содержать не менее 2 символов.'`. Если изменится текст ошибки (например, при смене локализации или обновлении Zod-схемы), тест упадёт. Рекомендуется использовать `data-testid` или проверять наличие элемента с ролью `alert`.
- **Проверка только имени:** Валидация пустой формы проверяет ошибку только для поля имени, но не для email, телефона или сообщения. Это может не выявить проблем с валидацией других полей. Для smoke-теста достаточно, но для полноты стоит добавить проверки.
- **Мок по паттерну URL:** `page.route('**/formspree.io/f/**')` предполагает, что endpoint содержит `/f/`. Если в продакшене используется другой путь (например, без `/f`), мок не сработает. Однако это тестовый мок, и в реальном проекте endpoint должен быть консистентным.
- **Валидные телефонные данные:** `validContactForm.phone` использует формат `+7 (999) 123-45-67`. Если валидация Zod изменится (например, будет требовать только цифры), тест может сломаться. Нужно синхронизировать с `lib/validation/contactFormSchema.ts`.

## Как проверить
1. Выполнить `npm run build` для сборки статики в папку `out/`.
2. Запустить `npm run test:e2e` (скрипт должен собрать тестовую конфигурацию с `NEXT_PUBLIC_FORMSPREE_ID=e2e_placeholder` и запустить сервер для `out/`).
3. Убедиться, что оба теста из `contact-form.spec.ts` проходят успешно.
4. Проверить, что при отправке пустой формы в консоли Playwright не появляется запрос на `formspree.io` (счётчик `formspreePosts` равен 0).
5. При валидных данных — появляется элемент с ролью `status`, содержащий текст `'Заявка отправлена. Мы свяжемся с вами по указанному email.'`.
