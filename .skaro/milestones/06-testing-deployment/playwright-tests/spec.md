# Спецификация: playwright-tests

## Контекст

Минимальное покрытие (smoke) для лендинга после статической сборки Next.js. Тестирование гарантирует работоспособность основных блоков сайта и корректность интеграции с внешним сервисом сбора заявок.

## User Scenarios

1. **CI/CD:** команда `npm run test:e2e` завершается успешно перед деплоем.
2. **Рендер:** на главной видны все ключевые секции по якорям навигации при прямой загрузке и скролле.
3. **Форма (Успех):** при заполнении валидных данных и отправке показывается сообщение об успехе.
4. **Форма (Ошибки):** пустая форма не отправляется; при сбое сервера Formspree выводится ошибка.

## Functional Requirements

- **FR-01: Секции главной.** Восемь секций с `id`, согласованными с [`lib/data/navigation.ts`](../../../../lib/data/navigation.ts): `#hero`, `#problems`, `#how-we-work`, `#cases`, `#products`, `#results`, `#about`, `#contact`. 
    - *Механика:* Для каждой секции выполняется принудительный скролл (`scrollIntoViewIfNeeded`), после чего проверяется видимость через `toBeVisible()`.
- **FR-02: Форма и мок Formspree.** 
    - **Позитивный сценарий:** Перехват POST на `formspree.io` (паттерн `**/formspree.io/f/**`), ответ 200 и JSON `{ "ok": true }`. Проверка сообщения об успехе (`role="status"`).
    - **Валидация:** Пустая отправка не должна инициировать сетевой запрос. Проверка наличия сообщений об ошибках (элементы с `role="alert"` или текст внутри контейнеров ошибок Zod).
    - **Негативный сценарий:** Перехват POST с ответом 500. Проверка отображения пользователю сообщения о системной ошибке.
- **FR-03: Окружение.** Прогон против статики в папке `out/`. 
    - *Запуск:* Использование встроенного в `playwright.config.ts` веб-сервера (`webServer`) с командой `npx serve out -p 3000`.

## Non-Functional Requirements

- **NFR-01:** Тестируется только статический артефакт сборки (`out/`).
- **NFR-02:** Viewport десктоп **1280×720**, проект Chromium.
- **NFR-03:** Загрузку картинок `next/image` в smoke не проверяем.
- **NFR-04:** Локаль интерфейса и сообщений assert — русский; в конфиге Playwright задано `locale: 'ru-RU'`.
- **NFR-05:** Плавность скролла (smooth scroll) не проверяется; достаточно факта наличия целевых ID на странице.

## Boundaries

- Нет unit-тестов компонентов, адаптивности (mobile/tablet), визуального регресса (screenshots), a11y и SEO-тегов.

## Acceptance Criteria

- [x] Все восемь секций найдены по перечисленным `id`, к ним выполнен скролл, и они видимы.
- [x] Пустая форма не инициирует POST запрос на Formspree; отображаются ошибки валидации (role="alert").
- [x] При валидных данных и моке Formspree (200 OK) показывается сообщение об успешной отправке.
- [x] При имитации ошибки сервера Formspree (500 Error) пользователь видит соответствующее уведомление об ошибке.
- [x] Тесты расположены в каталоге `/e2e/` в корне репозитория.
- [x] Прогон в headless режиме проходит успешно при использовании `npx serve` для папки `out/`.

## Примечания

- Для сценария Formspree в бандл должна попасть непустая `NEXT_PUBLIC_FORMSPREE_ID` на этапе сборки — используются скрипты `build:e2e` / `start:e2e` (см. `AI_NOTES.md`).
- Использование `npx serve` требует предварительной установки пакета `serve` или использования `npx` для временного запуска.
