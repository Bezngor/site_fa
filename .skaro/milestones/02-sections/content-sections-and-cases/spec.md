# Specification: content-sections-and-cases

## Context
Основной контент лендинга, включая интерактивный блок кейсов. Данный документ описывает структуру, поведение и визуальные требования к информационным блокам сайта.

## Requirements
- **Реализовать секции**:
    - **Hero**: Заголовок, подзаголовок, призыв к действию.
    - **Problems**: Описание 4 основных "болей" целевой аудитории.
    - **HowWeWork**: Описание 5 этапов процесса взаимодействия.
    - **Results**: Демонстрация достигнутых показателей.
    - **About**: Информация о компании.
    - **ProductLine**: Простой маркированный список категорий предлагаемых продуктов.
    - **CasesSection**: Интерактивный блок с примерами работ.
- **Механика CasesSection**:
    - Карточки с механизмом аккордеона (inline expand) для 3 кейсов.
    - **Независимое раскрытие**: Пользователь может открыть как один, так и все 3 кейса одновременно.
    - **Контент кейса**: Внутри раскрытого блока должны быть четко выделены разделы: "Задача", "Решение", "Результат" (используя списки и абзацы).
- **Визуализация**:
    - Использование библиотеки **Lucide-react** для иконок (с обязательным tree-shaking для оптимизации бандла).
    - Анимация: Использовать только **нативные CSS transitions** для плавного изменения высоты (max-height) и прозрачности (opacity) при раскрытии кейсов.
- **Данные**: Все тексты для кейсов должны импортироваться из `/lib/data/cases.ts`.

## Acceptance Criteria
- **Адаптивность**: Все секции корректно отображаются на мобильных устройствах, планшетах и десктопах.
- **Интерактивность кейсов**: Кейсы раскрываются и закрываются по клику без использования модальных окон. Состояние каждого кейса независимо.
- **Производительность**: Использование Server Components для всех информационных блоков (Hero, Problems, HowWeWork, Results, About, ProductLine). Секция CasesSection может использовать `'use client'` для управления состоянием аккордеона.
- **Стилизация**: Соответствие цветовой палитре (Primary: #1B2A4A, Accent: #F59E0B) и использование Tailwind CSS 4.
- **Контент**: Секция ProductLine реализована в виде лаконичного маркированного списка категорий без избыточной детализации.

## Open Questions (Resolved)
- ~~Какое поведение при открытии нескольких кейсов?~~ -> Независимое раскрытие.
- ~~Какая библиотека иконок?~~ -> Lucide-react.
- ~~Нужна ли сложная анимация?~~ -> Нет, только нативные CSS transitions.
- ~~Детализация ProductLine?~~ -> Простой маркированный список.

### AI_NOTES.md (Дополнение)
- Для реализации плавного раскрытия аккордеона на чистом CSS/Tailwind рекомендуется использовать связку `grid-rows-[0fr]` -> `grid-rows-[1fr]` или анимацию `max-height` с `overflow-hidden`.
- Убедиться, что иконки Lucide импортируются индивидуально (например, `import { Check } from 'lucide-react'`), чтобы избежать раздувания бандла.
