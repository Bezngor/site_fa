# Clarifications: content-sections-and-cases

## Question 1
Какое поведение ожидается при одновременном открытии нескольких кейсов в CasesSection?

*Context:* Это определяет, должен ли блок работать как классический аккордеон (только один открытый элемент) или позволять раскрывать все сразу.

**Options:**
- A) Классический аккордеон: при открытии нового кейса предыдущий автоматически закрывается
- B) Независимое раскрытие: пользователь может открыть все 3 кейса одновременно
- C) Сетка с расширением: кейс разворачивается на всю ширину ряда, сдвигая остальные

**Answer:**
Независимое раскрытие: пользователь может открыть все 3 кейса одновременно

## Question 2
Какую библиотеку иконок следует использовать для визуализации этапов и проблем?

*Context:* В спецификации упомянут Lucide-react, но архитектура требует строгого контроля размера бандла (<150 KB).

**Options:**
- A) Lucide-react (с использованием tree-shaking для минимизации размера)
- B) Инлайновые кастомные SVG-компоненты (максимальная производительность)
- C) Использование только стандартных символов и CSS-фигур

**Answer:**
Lucide-react (с использованием tree-shaking для минимизации размера)

## Question 4
Требуется ли анимация при раскрытии кейсов и переходе между секциями?

*Context:* Анимации влияют на bundle size и выбор библиотек (например, Framer Motion против CSS transitions).

**Options:**
- A) Только нативные CSS transitions (плавное изменение высоты и opacity)
- B) Легкая библиотека анимаций (например, clsx + Tailwind transitions)
- C) Без анимации: мгновенное появление контента для максимальной скорости

**Answer:**
Только нативные CSS transitions (плавное изменение высоты и opacity)

## Question 5
