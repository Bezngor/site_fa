# Specification: cta-form-and-integration

## Context
Финальный блок конверсии и техническая оптимизация проекта. Данный этап включает создание интерактивной формы обратной связи, интеграцию с внешним сервисом сбора заявок и финальную SEO-подготовку сайта.

## Requirements
- **Компонент `CtaFormSection`**:
    - Реализовать как Client Component (`'use client'`).
    - Поля формы: Имя, Email, Телефон, Сообщение (текстовое поле).
    - Валидация: Использование Zod для проверки схем.
    - Телефон: Простая проверка на наличие минимум 10 цифр (игнорируя спецсимволы).
    - Защита от спама: Добавить скрытое поле "honeypot" (совместимое с Formspree).
- **Интеграция с Formspree**:
    - Отправка данных методом POST на endpoint Formspree с использованием `NEXT_PUBLIC_FORMSPREE_ID`.
    - **Обработка ошибок**: Если `NEXT_PUBLIC_FORMSPREE_ID` отсутствует или API возвращает ошибку, вывести уведомление об ошибке и предложить прямую ссылку `mailto:`.
    - **Состояние успеха**: После успешной отправки скрыть форму и отобразить сообщение с благодарностью и кнопкой «Вернуться» (для сброса состояния).
- **SEO и Мета-данные**:
    - Настроить `robots.txt` в папке `public`.
    - Создать статический `public/sitemap.xml` вручную (так как сайт одностраничный).
    - Настроить мета-теги (Title, Description, OpenGraph) в корневом `layout.tsx`.

## Acceptance Criteria
- **Валидация**: Форма не отправляется, если поля не соответствуют правилам (валидный email, телефон >= 10 цифр).
- **Обработка состояний**:
    - Отображается индикатор загрузки (loading state) во время запроса.
    - При успехе: форма заменяется на Success Message.
    - При ошибке: появляется сообщение об ошибке с рабочей ссылкой `mailto:`.
- **Безопасность**: Наличие honeypot-поля для базовой фильтрации ботов.
- **Производительность**:
    - Итоговый Bundle size < 150 KB gzipped.
    - Lighthouse score по разделам Performance и SEO > 90.
- **SEO**: Файлы `sitemap.xml` и `robots.txt` доступны по прямым ссылкам.

## Технические детали реализации (на основе Q&A)
- **Валидация телефона**: Регулярное выражение, проверяющее `\d` (цифры), общее количество которых должно быть не менее 10.
- **Sitemap**: Поскольку проект является статическим лендингом, используется статический файл в `public/`, а не динамическая генерация через `sitemap.ts`.
- **Fallback**: В случае отсутствия ID ключа в окружении, форма переходит в режим "Direct Mail", предлагая пользователю отправить письмо через почтовый клиент.
