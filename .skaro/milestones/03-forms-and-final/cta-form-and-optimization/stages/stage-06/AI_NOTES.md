## Risks and limitations
- **Размер бандла**: Несмотря на оптимизации (tree-shaking, статическая генерация), добавление новых зависимостей может превысить лимит 150 KB. Рекомендуется регулярно запускать `npm run bundle-size` и анализировать bundle-анализер.
- **Зависимость от Formspree**: При недоступности Formspree форма использует fallback на mailto, что требует действий от пользователя. Это приемлемо для лендинга, но не идеально.
- **SSG и Client Components**: Next.js 15 корректно обрабатывает Client Components в SSG, но нужно убедиться, что `CtaFormSection` не использует серверные API (что и выполнено).

## How to verify
1. **Сборка**: `npm run build && next export` — должна завершиться без ошибок.
2. **Проверка bundle size**: `npm run bundle-size` — вывод должен показывать размер < 150 KB (например, `TotalSizeKB : 120.45`).
3. **Линтинг**: `npm run lint` и `npm run format:check` — должны проходить без ошибок.
4. **Рендеринг формы**: Откройте `out/index.html` в браузере, убедитесь, что секция CTA с формой отображается.
5. **Валидация**: Попробуйте отправить форму с невалидными данными (пустые поля, некорректный email, телефон менее 10 цифр) — должны появляться сообщения об ошибках.
6. **Отправка**: При наличии `NEXT_PUBLIC_FORMSPREE_ID` в `.env.local` форма должна отправляться (проверить в Formspree dashboard). При отсутствии — должен показываться fallback с ссылкой mailto.
7. **Lighthouse**: Запустите аудит в Chrome DevTools для `out/index.html` — Performance и SEO должны быть > 90.
8. **SEO-файлы**: Убедитесь, что `https://ваш-домен/robots.txt` и `https://ваш-домен/sitemap.xml` доступны и содержат корректные данные.
