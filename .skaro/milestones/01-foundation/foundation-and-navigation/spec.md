# Specification: foundation-and-navigation

## Context
Создать каркас Next.js 15 приложения и адаптивную навигацию для одностраничного лендинга консалтингового агентства FactoryAll. Проект ориентирован на высокую производительность (SSG) и строгое соответствие фирменному стилю.

## Requirements

### Tailwind CSS 4 конфигурация
Настроить цвета и базовые стили:
- **Цвета**:
  - `navy`: `#1B2A4A` (основной фон, текст)
  - `amber`: `#F59E0B` (акцент, кнопки CTA, выделения)
  - `white`: `#FFFFFF`
  - `gray-light`: `#F8F9FA` (фон секций)
- **Глобальные стили**:
  - В `globals.css` добавить `scroll-behavior: smooth` для плавной навигации по якорям.

### Header компонент (/components/layout/Header.tsx)
- **Поведение**: Sticky (фиксированный сверху, `z-50`), фон всегда полностью непрозрачный `navy` (#1B2A4A), без изменения прозрачности при скролле.
- **Логотип**: Текстовый «F!». Буква «F» — жирный шрифт, белый цвет; восклицательный знак «!» — янтарный цвет (#F59E0B). Использовать системные шрифты.
- **Десктопное меню** (якорные ссылки, белый текст):
  - «Проблемы» → `#problems`
  - «Как работаем» → `#how-we-work`
  - «Кейсы» → `#cases`
  - «Услуги» → `#products`
  - «О нас» → `#about`
  - **Индикация**: При скролле или наведении под активной ссылкой отображается линия (border-bottom или декоративный div), цвет текста остается неизменным.
  - **Кнопка «Записаться»**: Ссылка на `#cta`. Стиль: фон `amber`, текст `navy`, скругление углов (rounded).
- **Мобильное меню**:
  - Бургер-иконка (hamburger) для открытия.
  - Полноэкранное перекрытие (overlay) с тем же списком ссылок.
  - **Логика**: При клике на любую ссылку меню должно автоматически закрываться.
  - **Блокировка скролла**: При открытом мобильном меню необходимо устанавливать `overflow: hidden` для `body`, чтобы предотвратить прокрутку страницы на фоне.
- **Техническая реализация**: Использовать `'use client'`, так как компонент управляет состоянием (открытие меню, блокировка скролла).

### Данные (/lib/data/navigation.ts)
Экспортировать массив `navLinks` с полями: `label`, `href`. Это позволит централизованно управлять навигацией.

## Acceptance Criteria
1. **Фиксация**: Шапка всегда видна при скролле и имеет непрозрачный фон `#1B2A4A`.
2. **Мобильная навигация**:
   - Бургер-меню корректно открывается/закрывается.
   - При открытом меню скролл основной страницы заблокирован.
   - Клик по ссылке в мобильном меню закрывает его и перемещает пользователя к нужной секции.
3. **Плавность**: Переход по якорным ссылкам происходит плавно (`smooth scroll`).
4. **Визуализация**:
   - Логотип соответствует описанию (белая F, янтарный !).
   - Активные ссылки в меню подчеркиваются (визуальный индикатор).
5. **Адаптивность**: На экранах менее 768px десктопные ссылки скрываются, заменяясь на бургер-иконку.
6. **Производительность**: Использование стандартных HTML-тегов `nav`, `header` для SEO и доступности.

## AI_NOTES
- Для блокировки скролла в React рекомендуется использовать `useEffect`, который меняет `document.body.style.overflow` при изменении состояния `isOpen`.
- Для реализации активного состояния ссылок при скролле можно использовать `IntersectionObserver` в будущем, но на данном этапе достаточно базовой стилизации `hover` и подготовки структуры под `active`.
- Убедиться, что Z-index шапки выше, чем у любых других элементов страницы.
