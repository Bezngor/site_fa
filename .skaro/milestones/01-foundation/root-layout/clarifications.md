## Question 2
Какие именно шрифты Google Fonts необходимо подключить?

*Context:* Для обеспечения LCP < 2.5s и соблюдения дизайн-кода проекта нужно выбрать конкретные гарнитуры (например, Sans-serif для читаемости).

**Options:**
- A) Inter (основной) и Montserrat (заголовки) с поддержкой кириллицы.
- B) Roboto (основной) и Oswald (заголовки) для более индустриального вида.
- C) Системные шрифты (sans-serif) для максимальной производительности без загрузки внешних файлов.

**Answer:**
Inter (основной) и Montserrat (заголовки) с поддержкой кириллицы.

## Question 3

*Context:* Спецификация упоминает 'базовый body', но для одностраничного сайта важно понимать, фиксированы ли элементы навигации на уровне макета.

**Options:**
- A) Да, включить Header и Footer в layout.tsx, чтобы они были общими для всех (включая 404).
- B) Нет, Header и Footer добавить только в app/page.tsx как секции.
- C) Включить только Header в layout.tsx, Footer оставить внутри страницы.

**Answer:**
Да, включить Header и Footer в layout.tsx, чтобы они были общими для всех (включая 404).

## Question 4
Как обрабатывать ограничение 'max-w-7xl mx-auto' для секций с фоном на всю ширину?

*Context:* Если применить ограничение ширины к body или корневому контейнеру, фоновые цвета секций не будут растягиваться на весь экран.

**Options:**
- A) Применять max-w-7xl только к внутренним контейнерам каждой секции, оставляя обертку на 100% ширины.
- B) Оставить жесткое ограничение на body (весь контент в узкой колонке с полями по бокам).
- C) Использовать CSS-переменную для центрирования контента внутри layout.

**Answer:**
Применять max-w-7xl только к внутренним контейнерам каждой секции, оставляя обертку на 100% ширины.

## Question 5

- C) Строго темная тема (Dark mode) как основная визуальная концепция.
