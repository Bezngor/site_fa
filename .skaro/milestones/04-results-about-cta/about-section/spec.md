Вот обновленная спецификация для секции «О нас» (about-section), включающая все уточнения из Q&A.

```markdown
# Спецификация: about-section

## Контекст
Информационный блок о компании, предназначенный для формирования доверия у посетителя через описание ценностей и ключевых фактов.

## User Scenarios
1. **Посетитель:** Изучает информацию о компании, чтобы убедиться в ее экспертности и надежности.
2. **Посетитель:** Находит подтверждение ценностей компании и переходит к форме обратной связи.

## Functional Requirements
- **FR-01: Текстовый контент:** Отображение заголовка и краткого описания компании.
- **FR-02: Ключевые факты:** Список из 3-4 ключевых фактов или ценностей, представленных в виде иконок с пояснительным текстом.
- **FR-03: CTA Кнопка:** Кнопка «Связаться с нами», которая осуществляет плавный скролл к секции `CtaFormSection`.
- **FR-04: Централизованные данные:** Весь контент (тексты, иконки) должен импортироваться из объекта `about` в файле `lib/data/texts.ts`.

## Non-Functional Requirements
- **NFR-01: Responsive:** Адаптивная верстка. На мобильных устройствах элементы списка фактов выстраиваются вертикально.
- **NFR-02: Layout:** Одноколоночный макет (текст на всю ширину), так как блок с фото исключен.
- **NFR-03: Стилизация:** Использование фирменной палитры (Primary #1B2A4A, Accent #F59E0B).

## Boundaries (what is NOT included)
- Список конкретных сотрудников (Team members).
- Фотографии офиса или команды (согласно решению Q2).

## Acceptance Criteria
- [ ] Секция успешно интегрирована в основной лендинг.
- [ ] Данные подтягиваются из `lib/data/texts.ts`.
- [ ] Отсутствуют блоки для изображений; контент центрирован или распределен по всей ширине контейнера.
- [ ] Кнопка CTA корректно перенаправляет пользователя к форме контактов.
- [ ] Иконки отображаются корректно и соответствуют тематике фактов.

## Open Questions
- *Вопросов нет. Все неопределенности (фото, структура, CTA) разрешены в ходе Q&A.*
```

### AI_NOTES
- В `lib/data/texts.ts` необходимо создать структуру:
  ```typescript
  export const about = {
    title: "...",
    description: "...",
    features: [
      { icon: "IconName", title: "...", text: "..." },
      // 3-4 элемента
    ],
    ctaText: "Связаться с нами"
  };
  ```
- Для иконок рекомендуется использовать `lucide-react` (стандарт для современных Next.js проектов).
- Секция будет реализована как Server Component, так как данные статичны, за исключением кнопки скролла (которую можно сделать клиентским компонентом или использовать простой `<a>` якорь).
