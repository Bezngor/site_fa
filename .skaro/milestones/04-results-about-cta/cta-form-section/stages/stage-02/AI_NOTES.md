# AI_NOTES — Stage 2: Обновление Zod-схемы валидации и типов

## Что было сделано
- Проверена схема валидации `lib/validation/contactFormSchema.ts`: все поля (name, email, phone, message, _gotcha) соответствуют FR-01, сообщения об ошибках берутся из `texts.cta.form.validation`.
- Обновлён файл `types/index.ts`: тип `ContactFormData` переопределён как `Omit<ContactFormSchemaOutput, '_gotcha'>` чтобы избежать дублирования описания полей и обеспечить синхронизацию с схемой валидации.
- Добавлен импорт `ContactFormSchemaOutput` в `types/index.ts` (type-only import).

## Почему такой подход
- Схема валидации уже полностью соответствовала требованиям, поэтому не требовала изменений.
- Чтобы выполнить DoD "Экспорт типов без дублирования", `ContactFormData` был выведен из `ContactFormSchemaOutput` с исключением поля `_gotcha`. Это гарантирует, что при изменении схемы (например, добавлении trim или других трансформаций) тип `ContactFormData` автоматически отразит эти изменения, предотвращая рассинхронизацию.
- Использование `import type` обеспечивает отсутствие раннего импорта на этапе выполнения, что безопасно для типов.

## Файлы созданы/изменены
| Файл | Действие | Описание |

| `types/index.ts` | изменён | Заменено ручное определение `ContactFormData` на производный тип от `ContactFormSchemaOutput` с исключением `_gotcha`. Добавлен импорт типа. |

## Риски и ограничения
- Возможна циклическая зависимость, если `contactFormSchema.ts` в будущем начнёт импортировать из `types/index.ts`. Текущая архитектура этого не допускает.
- Если `ContactFormSchemaOutput` изменится (например, добавится новое поле), `ContactFormData` автоматически исключит его только если используется `Omit` с `_gotcha`. Но если добавится поле, которое должно быть в `ContactFormData`, потребуется ручная настройка. Однако в рамках текущих требований это не актуально.

## Соответствие инвариантам
- [x] Статическая генерация: изменения только в типах, не влияют на рантайм.
- [x] TypeScript: полная типизация, использование type-only import.
- [x] ESLint: код соответствует правилам (импорт в начале, camelCase и т.д.).

## Как проверить
1. Запустить `npx tsc --noEmit` — ошибок типов быть не должно.
2. Убедиться, что `ContactFormData` содержит только поля `name`, `email`, `phone`, `message` (без `_gotcha`).
3. Проверить, что другие типы в `types/index.ts` остались без изменений.
