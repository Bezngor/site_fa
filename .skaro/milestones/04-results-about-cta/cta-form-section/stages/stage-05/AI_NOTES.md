## Риски и ограничения
- Форма зависит от Formspree; при отсутствии `NEXT_PUBLIC_FORMSPREE_ID` будет использован fallback на `mailto:`.
- Для работы формы необходимо задать переменные окружения `NEXT_PUBLIC_FORMSPREE_ID` и опционально `NEXT_PUBLIC_CONTACT_EMAIL` в `.env.local`.
- Валидация только на стороне клиента; серверная валидация отсутствует (но Formspree выполняет свою проверку).

## Как проверить
1. Выполнить `npx tsc --noEmit` — типы должны быть корректны.
2. Выполнить `npx eslint components/sections/CtaFormSection.tsx lib/utils/formspree.ts lib/validation/contactFormSchema.ts` — ошибок быть не должно.
3. Выполнить `npx prettier --check components/sections/CtaFormSection.tsx lib/utils/formspree.ts lib/validation/contactFormSchema.ts` — форматирование должно соответствовать.
4. Выполнить `npm run build` — сборка должна завершиться успешно, без предупреждений о bundle size (бюджет 150 KB).
5. Вручную протестировать форму: заполнить все поля, отправить, убедиться в успешном сообщении; затем сымитировать ошибку Formspree (удалить `NEXT_PUBLIC_FORMSPREE_ID`) и проверить fallback с кнопкой «Отправить через почту».
