# Clarifications: cta-form-section

## Question 1
Какое поведение ожидается при срабатывании таймаута или ошибки Formspree?

*Context:* Необходимо определить, должен ли mailto: открываться автоматически или через дополнительное действие пользователя.

**Options:**
- A) Автоматически открывать почтовый клиент (window.location.href = mailto) при ошибке fetch
- B) Показывать сообщение об ошибке с кнопкой 'Отправить через почту'
- C) Просто выводить стандартное сообщение об ошибке без предложения mailto:

**Answer:**
Показывать сообщение об ошибке с кнопкой 'Отправить через почту'

## Question 2
Как должен быть реализован React Context для состояния формы?

*Context:* Спецификация требует Context, но для одной формы на странице это может быть избыточно по сравнению с локальным state.

**Options:**
- A) Создать FormProvider в отдельном файле для управления loading/error/success
- B) Использовать локальный useState внутри CtaFormSection (упрощение архитектуры)
- C) Использовать Context только если форма будет дублироваться в модальных окнах

**Answer:**
Использовать локальный useState внутри CtaFormSection (упрощение архитектуры)

## Question 3
Какой именно URL Formspree использовать для разработки и теста?

*Context:* Для работы fetch необходим корректный endpoint, иначе форма всегда будет уходить в fallback.

**Options:**
- A) Использовать заглушку (mock fetch) с искусственной задержкой 2 секунды
- B) Использовать переменную окружения NEXT_PUBLIC_FORMSPREE_URL
- C) Захардкодить тестовый ID (например, https://formspree.io/f/xdknqpgr)

**Answer:**
Использовать переменную окружения NEXT_PUBLIC_FORMSPREE_URL

## Question 4
Нужна ли визуальная индикация валидации полей в реальном времени?

*Context:* Zod позволяет проверять данные при вводе (onChange) или только при отправке (onSubmit).

**Options:**
- A) Валидация только при нажатии на кнопку 'Отправить'
- B) Валидация в реальном времени (onBlur/onChange) с отображением красной рамки
- C) Минималистичный вариант: только стандартные браузерные алерты

**Answer:**
Валидация только при нажатии на кнопку 'Отправить'

## Question 5
Какое сообщение об успехе должен видеть пользователь?

*Context:* После успешной отправки форма должна скрываться или заменяться контентом.

**Options:**
- A) Полная замена формы на компонент SuccessMessage с текстом благодарности
- B) Очистка полей формы и показ всплывающего уведомления (toast)
- C) Редирект на отдельную статическую страницу /thanks

**Answer:**
Полная замена формы на компонент SuccessMessage с текстом благодарности
