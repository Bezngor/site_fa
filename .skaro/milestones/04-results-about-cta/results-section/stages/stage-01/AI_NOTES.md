# AI_NOTES — Stage 1: Обновление данных и типов для секции результатов

## What was done
- Добавлен интерфейс `ResultMetric` в `types/index.ts` с полями `value`, `unit`, `label`, `icon?`.
- Добавлен экспорт `resultsMetrics` в `lib/data/texts.ts` с четырьмя метриками согласно спецификации.
- Импорт `ResultMetric` в `texts.ts` для типизации.

## Why this approach
- Тип `ResultMetric` позволяет строго типизировать данные метрик, что обеспечивает безопасность типов на этапе компиляции.
- Отдельный экспорт `resultsMetrics` (вне объекта `texts`) соответствует требованию загрузки метрик из `lib/data/texts.ts` и упрощает использование в компоненте.
- Поле `icon` сделано опциональным, так как в текущей спецификации данные метрик не содержат иконок; компонент может использовать иконки по умолчанию или логику назначения.

## Files created / modified
| File | Action | Description |

## Risks and limitations
- Отсутствие иконок в данных может потребовать дополнительной логики в компоненте (Stage 2) для отображения иконок по умолчанию или на основе label.
- Поле `icon` не используется в текущих данных, но оставлено для будущего расширения.
