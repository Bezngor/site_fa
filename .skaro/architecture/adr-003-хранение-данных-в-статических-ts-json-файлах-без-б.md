# ADR-003: Хранение данных в статических TS/JSON файлах без базы данных

**Status:** accepted
**Date:** 2026-03-21

## Context
Лендинг с фиксированным контентом (8 секций, 3 кейса) не требует динамических обновлений или пользовательских данных.

## Decision
Статические файлы в /lib/data/ (texts.ts, cases.json) для данных секций, кейсов и продуктов. TypeScript обеспечивает типизацию и валидацию на билд-тайме.

## Alternatives
1. **CMS (Strapi/Contntful):** Headless CMS — отвергнуто из-за необходимости серверного API и динамического рендера, противоречащего SSG.
2. **SQLite или JSON в public/**: Локальное хранение — отвергнуто из-за отсутствия нужды в runtime-доступе и риска bundle size.

## Consequences
- Positive: Нулевая задержка, нет зависимостей, типобезопасность.
- Negative: Обновления требуют redeploy.
- Risks: Ошибки в данных обнаруживаются только на билде.
