# ADR-001: Использовать Next.js 15 с App Router и Static Site Generation (SSG)

**Status:** accepted
**Date:** 2026-03-21

## Context
Для создания быстрого статического лендинг-пейджа с SEO-оптимизацией и минимальным bundle size требовался фреймворк, поддерживающий полную статическую генерацию без серверного рендера.

## Decision
Выбрано Next.js 15 с App Router и next export для SSG. Это обеспечивает генерацию HTML/CSS/JS на билд-тайме, отличную производительность, встроенную оптимизацию и совместимость с Vercel.

## Alternatives
1. **Gatsby.js:** Статический генератор на React — отвергнуто из-за более сложной настройки App Router-подобной структуры и большего bundle size по сравнению с Next.js.
2. **Vite + React (SSG плагины):** Легковесный билд — отвергнуто из-за отсутствия встроенной поддержки image optimization и metadata, что усложнило бы разработку.

## Consequences
- Positive: Максимальная скорость загрузки, SEO-дружественность, простота деплоя на Vercel.
- Negative: Нет динамического контента без redeploy.
- Risks: Изменения в контенте требуют полного переビルда.
